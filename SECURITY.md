# Security policy

Do not disclose vulnerabilities in public issues. Contact the repository owner privately through the security advisory feature on GitHub.

Include affected version, impact, reproduction steps, and a suggested mitigation. Do not include real API keys, access tokens, Firebase configs, user data, or signing material.

If a secret is pasted into chat, a ticket, a commit, or a log, treat it as compromised: revoke it immediately, issue a replacement, and inspect audit logs. Deleting the text is not sufficient.
